# Experiments

Crisp summary of what was run and what came out. Every number here is regenerated by the repo's
scripts; the full per-run output lives in [`reports/`](../reports).

**Common setup.** Backbone is timm `vit_small_patch16_224` with a 2-class head, trained two ways: a
**linear probe** (backbone frozen, head trained on cached features, CPU-friendly) and a **full
fine-tune** (whole model, 2 epochs, lr 1e-5). CIFAKE runs train on a 20k subset (18k train / 2k val)
on Apple-Silicon MPS and test on the full 20,000-image CIFAKE test split. The *operating threshold* is
the single score cutoff shared by the demo and every report, so a "fake" call means the same thing
everywhere.

## 1. Detection on CIFAKE — probe vs fine-tune

Headline accuracy, plus the false-positive cost at a fixed 95% detection rate.
Reports: [probe](../reports/report.md), [fine-tune](../reports/report_finetune.md).

| metric                         | Linear probe | Full fine-tune |
| ------------------------------ | ------------ | -------------- |
| ROC-AUC                        | 0.9766       | 0.9933         |
| accuracy @ 0.5                 | 0.9205       | 0.9612         |
| accuracy @ operating threshold | 0.9173       | 0.9576         |
| FPR @ TPR=0.95                 | 0.1161       | 0.0313         |
| operating threshold            | 0.4057       | 0.6883         |

**Takeaway.** Read FPR @ TPR=0.95 as the operating-point cost: to catch 95% of fakes, the fine-tune
wrongly flags 3.1% of real photos as fake, the probe 11.6%. The fine-tune roughly quarters the
false-accusation rate for about four points of accuracy.

## 2. False positives — which real images get called fake

At the operating threshold, **1,143** real images (probe) and **271** (fine-tune) are flagged as fake.
The 12 highest-scoring real images for each are copied to
[`reports/false_positives/`](../reports/false_positives) (`probe/` and `finetune/`) so you can see
*what* fools the detector, not just count it. A false accusation is the expensive error here.

## 3. Degradation — JPEG compression and resizing

ROC-AUC as 4,000 sampled test images are corrupted. A cliff means the detector leans on high-frequency
detail that the corruption destroys.

| condition    | probe AUC | fine-tune AUC |
| ------------ | --------- | ------------- |
| clean        | 0.9753    | 0.9938        |
| jpeg q=90    | 0.9743    | 0.9931        |
| jpeg q=70    | 0.9749    | 0.9928        |
| jpeg q=50    | 0.9620    | 0.9857        |
| jpeg q=30    | 0.9523    | 0.9784        |
| jpeg q=10    | 0.8529    | 0.8932        |
| resize ×0.75 | 0.9548    | 0.9670        |
| resize ×0.5  | 0.9239    | 0.9243        |
| resize ×0.25 | 0.8163    | 0.8104        |

**Takeaway.** Both hold through light JPEG and fall off under heavy compression and aggressive
downscaling. Note the last row: the fine-tune's big clean-image lead (0.994 vs 0.975) is gone at resize
×0.25 (0.810 vs 0.816) — extra same-generator accuracy buys almost no robustness once the image is
degraded.

## 4. Cross-generator transfer (GenImage) — the real test

Train a probe on **one** generator (shared real set `bitmind/bm-real`), then test on generators it
never saw. 250 real + 250 fake per test set; the real distribution is identical across train and test,
so the gap is generator shift, not domain shift. Run with
[`scripts/cross_generator_experiment.py`](../scripts/cross_generator_experiment.py). Reports:
[midjourney](../reports/cross_generator_midjourney.md), [biggan](../reports/cross_generator_biggan.md).

**Trained on MidJourney:**

| test generator        | ROC-AUC | FPR @ TPR=0.95 |
| --------------------- | ------- | -------------- |
| midjourney (same-gen) | 0.9190  | 0.2840         |
| biggan (unseen)       | 0.9914  | 0.0280         |
| adm (unseen)          | 0.9997  | 0.0000         |
| glide (unseen)        | 0.8956  | 0.5240         |

Mean unseen AUC **0.9622** — actually *higher* than same-gen (0.9190).

**Trained on BigGAN:**

| test generator      | ROC-AUC | FPR @ TPR=0.95 |
| ------------------- | ------- | -------------- |
| biggan (same-gen)   | 0.9319  | 0.1760         |
| midjourney (unseen) | 0.8351  | 0.7480         |
| adm (unseen)        | 0.8763  | 0.5160         |
| glide (unseen)      | 0.9101  | 0.4320         |

Mean unseen AUC **0.8738** — a 0.058 drop from same-gen, and on unseen MidJourney the operating point
collapses (FPR 0.748: catching 95% of fakes would flag roughly three of every four real photos).

**Takeaway.** Transfer is *asymmetric*. Train on a strong modern generator (MidJourney) and older
generators with blatant artifacts are easy; train on an older one (BigGAN) and modern fakes slip
through. A detector is only as good as the hardest generator it trained against — which a
same-generator accuracy number hides completely.
